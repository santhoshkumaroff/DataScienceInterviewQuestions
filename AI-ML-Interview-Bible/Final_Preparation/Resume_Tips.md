# Resume Tips for AI/ML Roles

## How to Create a Resume That Gets You Interviews

Your resume is your ticket to the interview. In a sea of applications, it needs to stand out. Here's exactly how to build one that recruiters love and ATS systems can read.

---

# Part 1: What to Include (and What NOT to Include)

## MUST Include

| Section | Why It Matters |
|---------|---------------|
| Contact Info (Email, Phone, LinkedIn, GitHub) | So they can reach you |
| Education (College, Degree, GPA if > 7.5/10) | Shows your foundation |
| Technical Skills | ATS keywords, quick scan |
| Projects (2-4 relevant ML projects) | PROVES you can DO the work |
| Experience / Internships | Real-world application |
| Achievements / Awards | Differentiation |
| Certifications (if relevant) | Extra validation |

## DO NOT Include

| Don't Include | Why Not |
|--------------|---------|
| Photo | Unprofessional, wastes space, can cause bias |
| Age, Gender, Marital Status | Irrelevant and can cause bias |
| Multiple Addresses | Just city and state is enough |
| Objective Statement | Takes space, rarely read. Use Summary instead |
| Irrelevant Hobbies | Unless they add value (e.g., Kaggle) |
| Too Many Colors / Graphics | ATS systems can't read them |
| Tables / Columns | ATS systems read left-to-right, top-to-bottom |
| Fancy Fonts | Stick to Arial, Calibri, Times New Roman |
| "References Available on Request" | Everyone knows this. Wastes space. |
| Salary History / Expectations | Not needed on resume |

# Part 2: How to Describe Projects (THE MOST IMPORTANT SECTION)

## The 3-Line Formula for Each Project

| Line | What to Cover | Example |
|------|--------------|---------|
| Line 1 | **What** you built + **Tech stack** | Built a crop disease detection system using CNN in TensorFlow |
| Line 2 | **How** you built it (key steps) | Used transfer learning with ResNet50, implemented data augmentation, achieved 94% accuracy |
| Line 3 | **Impact** (numbers, results, outcome) | Reduced manual inspection time by 80%, deployed as a web app using Flask |

## Project Description Template

**Project Name | [Timeline]**

- Built [what] using [tech stack] to solve [problem]
- Implemented [key techniques/features] which resulted in [specific outcome]
- Achieved [metric] and deployed using [tools]

## Examples of Strong Project Descriptions

**Good:**
> Built a sentiment analysis model using LSTM and Word2Vec embeddings trained on 50,000 movie reviews with 89% accuracy.

**Better:**
> Developed a real-time sentiment analysis system for Twitter using LSTM networks (TensorFlow). Preprocessed 100K+ tweets with NLTK, implemented Word2Vec embeddings, and achieved 89% F1-score. Deployed as a REST API with Flask on AWS EC2.

**Best:**
> End-to-End Sentiment Analysis System: Built a production-ready NLP pipeline processing 100K+ tweets using LSTM with Word2Vec (89% F1-score). Implemented custom preprocessing with NLTK, real-time inference via Flask API, and deployed on AWS EC2 with Docker. Used by 200+ users within first month.

# Part 3: Keywords for ATS (Applicant Tracking Systems)

## What is ATS?
Companies use software to scan resumes for relevant keywords before a human ever sees them. If your resume doesn't have the right keywords, it gets rejected automatically.

## Top AI/ML Keywords

**Technical Skills:**
Python, R, SQL, TensorFlow, PyTorch, Keras, Scikit-learn, Pandas, NumPy, Matplotlib, Seaborn, OpenCV, NLP, Computer Vision, Deep Learning, Machine Learning, Neural Networks, CNN, RNN, LSTM, Transformer, BERT, GPT, XGBoost, LightGBM, Random Forest, SVM, K-Means, PCA, Docker, Kubernetes, AWS, GCP, Azure, Flask, FastAPI, Git, Linux

**Concepts:**
Data Preprocessing, Feature Engineering, Model Training, Hyperparameter Tuning, Cross-Validation, Overfitting, Underfitting, Regularization, Gradient Descent, Backpropagation, Evaluation Metrics, Precision, Recall, F1-Score, ROC-AUC, Confusion Matrix, A/B Testing

**Soft Skills (matter for ATS too):**
Problem Solving, Critical Thinking, Communication, Team Collaboration, Time Management, Analytical Skills, Research

## How to Use Keywords
1. Read 10 job descriptions for your target role
2. List the top 20 skills that appear most frequently
3. Include them naturally in your resume (in skills section AND in project descriptions)
4. Don't just list them - show them in context

# Part 4: Formatting Tips

## Layout Order (Recommended)

1. **Name + Contact** (top, centered or left-aligned)
2. **Professional Summary** (2-3 lines)
3. **Technical Skills** (bulleted list by category)
4. **Experience / Internships** (reverse chronological)
5. **Projects** (2-4 projects)
6. **Education** (college, degree, graduation year, GPA)
7. **Certifications / Achievements** (1-2 lines each)
8. **Additional** (languages, interests - optional)

## Formatting Rules

| Element | Recommendation |
|---------|---------------|
| Font | Arial, Calibri, Times New Roman (10-12pt) |
| Page Length | 1 page for freshers, 1-2 pages for 2+ years |
| Margins | 0.5 to 1 inch |
| File Format | PDF (unless they ask for Word) |
| File Name | FirstName_LastName_Resume.pdf |
| Bullet Points | Use the same symbol throughout |
| Bold | Use for section headers and project names |
| Colors | Only ONE accent color if any (dark blue/gray) |

## Once You Have a Draft

**Test with real job descriptions.** Copy-paste a job description and highlight which keywords from it are already in your resume. If there are too many missing, go back and add them contextually.

# Part 5: Quantifying Achievements

## Why Numbers Matter
Recruiters love numbers. They're concrete, memorable, and prove impact.

## How to Add Numbers to Everything

| Instead of This | Write This |
|----------------|-----------|
| Built a classification model | Built a classification model with 94% accuracy on 10K+ images |
| Worked on data cleaning | Cleaned 50GB of data using Pandas, reducing null values from 15% to 0.5% |
| Internship at XYZ | Developed 3 dashboards used by 50+ team members weekly |
| Participated in hackathon | Placed top 10 among 500+ teams in national hackathon |
| Contributed to open source | Fixed 5 bugs in TensorFlow documentation over 3 months |
| Created a chatbot | Built a customer support chatbot handling 200+ queries/day |

## Metrics You Can Use

**Model Performance:** accuracy, precision, recall, F1-score, RMSE, MAE, R-squared, latency
**Scale:** number of data points, features, users, requests, images, documents
**Impact:** time saved, cost reduced, revenue increased, efficiency improved
**Speed:** training time, inference time, response time
**Size:** team size, project timeline, dataset size

# Part 6: Skills Section Layout

## How to Organize Your Skills

**Category-Based Layout (Recommended):**

**Technical Skills:**
- **Languages:** Python, SQL, R, C++
- **ML/DL:** TensorFlow, PyTorch, Scikit-learn, Keras, OpenCV
- **Data Processing:** Pandas, NumPy, NLTK, BeautifulSoup
- **Visualization:** Matplotlib, Seaborn, Tableau
- **MLOps & Tools:** Docker, Git, Linux, Flask, AWS
- **Databases:** MySQL, MongoDB

## Tips for Skills Section
1. List skills in order of relevance to the target role
2. Don't use progress bars or "80% proficient" - they're meaningless
3. Only list skills you can confidently discuss in an interview
4. Include both hard skills (Python, TensorFlow) and tools (Git, Docker)
5. Update this section for each job application

# Part 7: Education Section

## What to Include

**Format:**
> **Degree** | College Name | City | Graduation Year
> - GPA: X/10 (if > 7.5)
> - Relevant Coursework: ML, Deep Learning, Statistics, Linear Algebra, Data Mining
> - Honors / Awards: Dean's List, Academic Scholarship

## Tips
- Include GPA only if it's good (> 7.5/10 or > 3.5/4)
- List relevant coursework that matches the job
- Mention thesis or capstone project with a brief description
- Include relevant extracurriculars (AI club, hackathons)

**If you're from a non-CS background:**
- Highlight ML courses you've taken (Coursera, Udacity, etc.)
- Mention math background (statistics, linear algebra) as a strength
- Link projects to show practical application

# Part 9: ATS-Friendly Tips

## How to Make Your Resume ATS-Friendly

1. **Use standard section headers** - "Experience", "Education", "Skills", "Projects" (not creative names)

2. **NO tables, columns, text boxes, graphics** - ATS reads top-to-bottom, left-to-right

3. **NO headers/footers** - ATS often misses text in headers/footers

4. **Use standard fonts** - Arial, Calibri, Times New Roman, Helvetica

5. **Save as PDF** (unless they ask for .docx) - PDF preserves formatting

6. **Don't use special characters** - Bullets, arrows, symbols may not render in ATS

7. **Spell out acronyms** - First use: "Convolutional Neural Network (CNN)", then "CNN"

8. **Include keywords from the job description** - Match their language

9. **Use .pdf in filename** - "FirstName_LastName_Resume.pdf"

10. **Test with an ATS simulator** - Use Jobscan or similar before submitting

## Quick ATS Test
Copy your resume text and paste it into Notepad. If it looks clean and readable, ATS can read it too. If you see scrambled text or missing content, reformat.
